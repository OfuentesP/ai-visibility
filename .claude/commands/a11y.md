# Accessibility Contrast Audit

Audit all TSX files in `src/` for WCAG AA contrast violations and fix them.

## Step 1 — Collect files

Use Bash to find all `.tsx` files under `src/`:
```
find src -name "*.tsx" -not -path "*/node_modules/*"
```

## Step 2 — Extract color pairs

For each file, read it and identify every Tailwind text + background color combination present in the same element or parent/child relationship. Focus on:
- `text-*` paired with `bg-*`
- `text-*` on elements inside containers with `bg-*`
- Placeholder colors (`placeholder-*`)
- Border-only text on dark backgrounds

## Step 3 — Evaluate contrast using this Tailwind palette reference

Use these approximate luminance values to compute WCAG contrast ratio = (L1 + 0.05) / (L2 + 0.05) where L1 > L2.

**WCAG AA requires:** ≥ 4.5:1 for normal text, ≥ 3:1 for large text (≥18px or bold ≥14px).

Tailwind slate approximate hex values:
- slate-950: #020617  (L≈0.001)
- slate-900: #0f172a  (L≈0.005)
- slate-800: #1e293b  (L≈0.013)
- slate-700: #334155  (L≈0.030)
- slate-600: #475569  (L≈0.056)
- slate-500: #64748b  (L≈0.109)
- slate-400: #94a3b8  (L≈0.212)
- slate-300: #cbd5e1  (L≈0.371)
- slate-200: #e2e8f0  (L≈0.527)
- slate-100: #f1f5f9  (L≈0.699)
- white: #ffffff      (L≈1.000)

## Step 4 — Report violations

List every violation as:
```
FILE:LINE — text-{color} on bg-{color} → ratio X.X:1 (need ≥4.5:1)
```

Pay special attention to:
- `text-slate-700` or darker on `bg-slate-950/slate-900` — likely fails
- `text-slate-600` on dark backgrounds — borderline
- `text-slate-800` used as visible label text — probably fails on dark bg
- Placeholder text (`placeholder-slate-600`) — informational, note but don't fix unless extreme
- Small `text-[10px]` or `text-xs` — stricter ratio needed

## Step 5 — Fix violations

For each violation, edit the file and upgrade the text color to the minimum passing shade. Prefer the smallest change:
- `text-slate-700` → `text-slate-400` on dark backgrounds
- `text-slate-800` → `text-slate-500` minimum, prefer `text-slate-400`
- `text-slate-600` → `text-slate-400` on slate-900/950
- Keep semantic meaning: muted text should stay muted (e.g., slate-500 not white)

Do NOT change:
- Decorative elements (dividers, borders, background shapes)
- `opacity-*` classes used for visual layering
- Colors already passing

## Step 6 — Summary

After all fixes, output a table:
| File | Original | Fixed | Ratio before → after |
|------|----------|-------|----------------------|

Then list any remaining borderline cases (3.5–4.4:1) as warnings for manual review.
